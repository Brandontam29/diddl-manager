- Hero: centered text, max-width `max-w-2xl mx-auto`, `text-center`
- CTA row: `flex gap-4 justify-center`
- Feature cards: `grid grid-cols-1 sm:grid-cols-3 gap-4 max-w-4xl mx-auto`

### Header Updates (All `/app` Pages)

```
+-----------------------------------------------------+
|  [<- back/logo]  Diddl Manager     [Sign In / Avatar]|
+-----------------------------------------------------+
```

- Guest state: "Sign In" button, `variant="outline" size="sm"`
- Authed state: Clerk `UserButton` component (avatar with dropdown)
- UserButton dropdown includes: "Profile" link (Phase 4), "Sign out" action
- Header height: 64px (unchanged from Phase 1/2)

### Landing Page Header (Different from App Header)

- No sidebar, no catalog navigation
- Logo/title left-aligned, "Sign In" button right-aligned
- Same 64px height, `bg-background border-b`

### Store Switching Shimmer (In-Place)

When auth state changes from guest to authed (after sign-in + migration):

```
+-----------------------------------------------------+
|  Header (now shows avatar)                          |
+--------------+--------------------------------------+
|  Sidebar     |  [Shimmer blocks]                    |
|  (256px)     |  [Shimmer blocks]                    |
|              |  [Shimmer blocks]                    |
+--------------+--------------------------------------+
```

- Shimmer replaces the list card grid area only (sidebar and header remain)
- Shimmer blocks: `bg-muted animate-pulse rounded-lg` matching card dimensions
- 3 shimmer blocks in a row matching the list card grid layout
- Duration: visible for 1-2 seconds during migration, then replaced by real data

Inherited from Phase 1/2 shadcn-svelte stone theme. Uses CSS custom properties defined in `layout.css`.

| Role            | CSS Variable    | Usage                                               |
| --------------- | --------------- | --------------------------------------------------- |
| Dominant (60%)  | `--background`  | Page background, landing page background            |

| Accent (10%)    | `--primary`     | "Sign In" CTA on landing, "Try as Guest" button     |

**Accent (`--primary` / stone-900 light, stone-100 dark) reserved for:**

1. Landing page "Try as Guest" button (primary CTA, `variant="default"`)
2. Header "Sign In" button uses `variant="outline"` (not accent-filled)
3. Active sidebar range row background (inherited)

**Clerk modal theming:**

- Clerk appearance object must map to the app's CSS custom properties
- Background: `--card` value
- Primary button: `--primary` value
- Text: `--foreground` value
- Input borders: `--input` value
- The Clerk modal overlay uses `bg-black/50` backdrop (standard Clerk behavior, consistent with shadcn Dialog overlay)

**Shimmer color:**

- Shimmer blocks: `bg-muted` with `animate-pulse` (uses Tailwind's built-in pulse animation)

**Upgrade prompt styling:**

- Background: `bg-muted/50` (subtle, non-intrusive)
- Text: `text-muted-foreground`
- Link/CTA within prompt: `text-primary` with `underline underline-offset-4`

## Interaction States

### Header Sign-In Button (Guest State)

- Default: `variant="outline" size="sm"`, label "Sign In"
- Hover: standard outline button hover (border darkens)
- Click: calls `clerk.openSignIn()` to open Clerk modal overlay
- Visible only when `clerkStore.currentUser === null` and `clerkStore.isClerkLoaded === true`

### Header UserButton (Authed State)

- Rendered via Clerk's `UserButton` mount API
- Shows user avatar (32px circle)
- Click: opens Clerk-managed dropdown with profile link and sign-out
- Sign-out: redirects to `/` (D-05)

### Landing Page CTAs

- "Try as Guest" button: `variant="default" size="lg"` (accent/primary fill)
  - Click: navigates to `/app` (enters guest mode, no auth required)
- "Sign In" button: `variant="outline" size="lg"`
  - Click: calls `clerk.openSignIn()` to open Clerk modal

### Clerk Modal Overlay

- Overlay: `bg-black/50` backdrop covering full viewport
- Modal: centered, Clerk-managed layout
- Themed with app colors (see Color section — Clerk modal theming)
- Close: click outside or X button (Clerk default behavior)
- On successful sign-in: modal closes, Clerk listener fires, store switches

### Store Switch Transition (D-06)

1. User signs in via Clerk modal (on any `/app` page)
2. Clerk listener detects `currentUser` change from `null` to user object
3. Shimmer grid replaces list content area (sidebar + header remain visible)
4. Migration action fires in background (sends localStorage data to Convex)
5. On success: shimmer replaced by real Convex data, success toast appears
6. On failure: retry up to 3 times with 1s, 2s, 4s backoff; show retry toast each time
7. On final failure: show failure toast, revert to GuestListStore (localStorage intact)
8. No full page reload at any point

### Shimmer Grid

- 3 shimmer blocks in `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3` matching list card grid
- Each block: `h-32 bg-muted animate-pulse rounded-lg`
- Screen reader: `aria-label="Loading your collection..."`
- `role="status"` on the shimmer container

### Upgrade Prompts (D-13)

- **2nd list attempt (guest):** Inline text below the disabled "Create List" button: "Sign up to create more lists" with "Sign up" as a clickable link that opens `clerk.openSignUp()`
- **Lists page footer (guest):** Centered muted text at bottom of lists overview: "Sign up to sync across devices" with "Sign up" as a clickable link
- Both prompts: `text-sm text-muted-foreground`, link portion `text-primary underline underline-offset-4 cursor-pointer`
- Both prompts: hidden when user is authed

### Migration Toasts

- Success: `toast.success("Your collection has been saved to your account.")` — auto-dismiss after 4 seconds
- Retry: `toast.info("Could not save collection. Retrying...")` — auto-dismiss after 3 seconds
- Final failure: `toast.warning("Data safe in browser. We'll try again next time.")` — persists until dismissed
- All toasts use svelte-sonner (already installed)

### Sign-Out Flow

- User clicks "Sign out" in Clerk UserButton dropdown
- Clerk handles sign-out, fires listener with `currentUser: null`
- App redirects to `/` (landing page) per D-05
- On next visit to `/app`, GuestListStore is created (localStorage data still present if any)

## Accessibility Notes

- Landing page CTAs must be keyboard-accessible (tab + enter/space)
- Clerk modal handles its own focus trap and keyboard navigation
- Header "Sign In" button: standard `<button>` element, keyboard-accessible
- Shimmer grid: `role="status"` with `aria-label="Loading your collection..."` for screen readers
- Upgrade prompt links: must be `<button>` elements (not `<a>`) since they trigger JS actions (opening Clerk modal), with `role="link"` styling but proper button semantics
- UserButton area: Clerk manages its own ARIA attributes
- Toast announcements: `role="status"` (svelte-sonner default)
- Landing page feature cards: decorative only (no interactive elements), icons have `aria-hidden="true"`
- All interactive elements: minimum 44px touch target

# Phase 2 — UI Design Contract: Guest List Mode

> Visual and interaction contract for the list management experience. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

| Property          | Value                                           |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage                                                    |
| ----- | ----- | -------------------------------------------------------- |
| xs    | 4px   | Icon gaps, color swatch spacing, stepper button padding  |
| sm    | 8px   | Card inner gaps, toolbar action spacing, checkbox offset |
| md    | 16px  | Card padding, dialog field gaps, grid gap                |
| lg    | 24px  | Section padding, toolbar vertical padding                |
| xl    | 32px  | Layout column gap (sidebar to main content)              |
| 2xl   | 48px  | Page-level top/bottom padding                            |
| 3xl   | 64px  | Not used in Phase 2                                      |

Exceptions:

- Touch targets for toolbar actions and quantity stepper buttons: minimum 44px hit area (accessibility)
- Color palette swatches: 28px diameter (7 \* 4px, fits 8-point grid)
- Catalog card image area: fixed 160px height (inherited from Phase 1)

## Layout

### Lists Overview Page (`/app/lists`)

**Primary focal point:** The list card grid. When lists exist, the user's eye should land on the card grid first. When no lists exist, the empty state CTA ("Create List") is the focal point.

```
+-----------------------------------------------------+
|  Header (full-width, 64px)                          |
+-----------------------------------------------------+
|  Page heading: "My Lists"         [Create List btn] |
|                                                     |
|  +---------------+  +---------------+               |
|  | ListCard      |  | ListCard      |  ...          |
|  | color bar     |  | (guest: max 1)|               |
|  | name          |  |               |               |
|  | completion %  |  |               |               |
|  | item count    |  |               |               |
|  +---------------+  +---------------+               |
+-----------------------------------------------------+
```

- List cards: responsive grid `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`
- Gap: `gap-4` (16px)
- "Create List" button: disabled with tooltip for guests who already have 1 list

### List Detail Page (`/app/lists/[id]`)

**Primary focal point:** The list item grid. The toolbar and sidebar serve as supporting navigation; the grid of owned items is the primary content area the user interacts with.

```
+-----------------------------------------------------+
|  Header (full-width, 64px)                          |
+--------------+--------------------------------------+
|  ListSidebar |  Toolbar                             |
|  (256px)     |  [Show Unowned] [Filter] [Condition] |
|              |  (visible when items selected:       |
|  Category    |   Duplicate | Complete | Incomplete  |
|  tree with   |   | Set Condition | Remove)          |
|  owned/total |--------------------------------------+
|  counts      |  List Item Grid                      |
|              |  (cards with image, name, qty,       |
|              |   checkbox for multi-select)          |
|              |                                      |
+--------------+--------------------------------------+
```

- Sidebar: `w-64` (256px), sticky, full viewport height, independent scroll (same as catalog)
- Main content: `flex-1`, min-width 0, scrollable
- Toolbar: sticky at top of main content area, `z-10`

### Catalog Page Updates (Select Mode)

```
+-----------------------------------------------------+
|  Header (64px)                                      |
+--------------+--------------------------------------+
|  Sidebar     |  [Select Mode toggle]                |
|  (256px)     |  When active:                        |
|              |  [X selected] [Add to List] [Done Selecting] |
|              |--------------------------------------+
|              |  Catalog Grid (cards now show         |
|              |   checkboxes in top-left corner)      |
+--------------+--------------------------------------+
```

- Select mode toggled via button in catalog page header area
- Floating action bar appears when 1+ items selected

## Typography

Inherited from Phase 1. No new sizes or weights.

| Role    | Size | Weight         | Line Height | Tailwind class            |
| ------- | ---- | -------------- | ----------- | ------------------------- |
| Body    | 14px | 400 (regular)  | 1.5         | `text-sm`                 |
| Label   | 12px | 600 (semibold) | 1.4         | `text-xs font-semibold`   |
| Heading | 16px | 600 (semibold) | 1.25        | `text-base font-semibold` |
| Display | 20px | 600 (semibold) | 1.2         | `text-xl font-semibold`   |

**Two weights only:** 400 (regular) and 600 (semibold).

**Phase 2 usage mapping:**

- List card name: `text-sm font-semibold`
- List card item count: `text-xs` (regular, muted)
- List detail page heading (list name): `text-xl font-semibold`
- Toolbar action labels: `text-sm`
- Toolbar selected count: `text-sm font-semibold`
- Quantity stepper count: `text-xs font-semibold` (centered, min-width 1.5rem)
- Dialog field labels: `text-sm font-semibold`
- Dialog input text: `text-sm`
- Completion percentage on list card: `text-xs font-semibold`
- Completion percentage on detail page: `text-base font-semibold`
- Sidebar owned/total counts: `text-xs` (regular, muted)
- "Show unowned" label: `text-sm`
- Empty state heading: `text-base font-semibold`
- Empty state body: `text-sm` (regular, muted)
- Toast messages: inherits from svelte-sonner defaults

## Color

Inherited from Phase 1 shadcn-svelte stone theme. Uses CSS custom properties defined in `layout.css`.

| Role            | CSS Variable    | Usage                                                         |
| --------------- | --------------- | ------------------------------------------------------------- |
| Dominant (60%)  | `--background`  | Page background, sidebar background                           |
| Secondary (30%) | `--card`        | Card surfaces, dialog surfaces, toolbar background            |
| Accent (10%)    | `--primary`     | Primary action buttons, active sidebar row, select mode badge |
| Destructive     | `--destructive` | Delete list button, remove items action, destructive dialogs  |

**Accent (`--primary` / stone-900) reserved for:**

1. "Create List" button (primary CTA)
2. "Add selected to list" button in catalog select mode
3. Active sidebar range row background (inherited from Phase 1)
4. Selected item count badge in toolbar

**Destructive (`--destructive`) reserved for:**

1. "Delete List" button in list overview
2. "Remove" action in toolbar when items are selected
3. Delete confirmation dialog action button

**List color indicators:** User-chosen preset palette colors (see Color Palette section below). These are decorative per-list identifiers and do not participate in the 60/30/10 system color split.

**Completion badge colors (reused from Phase 1):**

- 0-33%: `bg-red-50 text-red-600`
- 34-66%: `bg-amber-50 text-amber-600`
- 67-99%: `bg-blue-50 text-blue-600`
- 100%: `bg-emerald-50 text-emerald-700`

**Unowned items visual treatment (Claude's discretion, D-08):**

- Unowned catalog items in "show unowned" mode: `opacity-50` with dashed border (`border-dashed border-muted-foreground/30`)
- This distinguishes owned (solid, full opacity) from unowned (faded, dashed) without adding new semantic colors

**Selected card treatment:**

- Selected: `ring-2 ring-primary` border highlight
- Checkbox in top-left: shadcn Checkbox component, visible always in list detail, visible only in select mode on catalog

## Interaction States

### List Card (Overview Page)

- Default: `bg-card` surface, `border` border, 4px color bar at top
- Hover: `shadow-md transition-shadow`
- Click: navigates to `/app/lists/[id]`
- Edit icon button (top-right): `variant="ghost" size="icon"`, `aria-label="Edit {listName}"`, opens EditListDialog
- Delete icon button (top-right): `variant="ghost" size="icon"` with `text-destructive` on hover, `aria-label="Delete {listName}"`, opens DeleteListDialog

### List Item Card (Detail Page)

- Default: `bg-card` surface, `border` border, full opacity
- Hover: `shadow-md transition-shadow`
- Selected: `ring-2 ring-primary` border highlight
- Checkbox (top-left): always visible, toggles selection state
- Quantity stepper (bottom-left): inline, buttons have `variant="ghost" size="icon"` with 24px height

### Unowned Item Card (Show Unowned Mode)

- Same card layout as list item card
- Visual distinction: `opacity-50 border-dashed`
- No checkbox, no quantity stepper
- Click or dedicated "+" button: adds item to list via `listStore.addItems()`
- On add: card transitions from unowned (faded) to owned (full opacity, solid border) without page refresh

### Toolbar (List Detail)

- Default state (no selection): shows "Show Unowned" toggle, "Missing Only" filter toggle, list name, completion badge
- Selection state (1+ items selected): shows selection count badge + action buttons (Duplicate, Complete, Incomplete, Condition dropdown, Remove)
- Transition: toolbar content swaps with subtle crossfade when selection changes between 0 and 1+
- Sticky: remains at top of main content area while scrolling grid
- Background: `bg-card` with `border-b` bottom border

### Condition Dropdown (Toolbar)

- Trigger: "Condition" button with chevron-down icon
- Menu items: Mint, Near Mint, Good, Poor, Damaged
- Selecting applies condition to all selected items
- No checkmark indicator (applies immediately, not a toggle)

### Create List Dialog

- Trigger: "Create List" button
- Fields: Name text input (required), Color palette (radio-like selection)
- Submit: "Create List" button, `variant="default"` (primary)
- Cancel: close dialog via X button or click outside
- Validation: Name must be non-empty; show inline error "Name is required" below input
- Guest guard: `GuestListStore.createList()` throws if 1 list already exists; show toast

### Edit List Dialog

- Trigger: edit icon on list card
- Fields: Name text input (pre-populated), Color palette (pre-selected)
- Submit: "Save Changes" button, `variant="default"` (primary)
- Cancel: close dialog via X button or click outside

### Delete List Dialog

- Trigger: delete icon on list card
- AlertDialog with destructive action
- Action button: "Delete", `variant="destructive"`
- Cancel button: "Keep List"
- Body dynamically includes list name and item count

### Select Mode (Catalog Page)

- Toggle: "Select" button in catalog page header area
- Active: button becomes "Done Selecting" (`variant="outline"`), toolbar appears below header
- Cards: show checkbox in top-left corner (overlaid on image area)
- "Add to List" button: appears when 1+ items selected, `variant="default"` (primary)
- For guests (1 list): adds directly, no picker needed
- For authed (multiple lists): DropdownMenu list picker (D-07)
- Exit: "Done Selecting" button clears selection and exits select mode

### Show Unowned Toggle

- Toggle component in toolbar
- Off (default): grid shows only items in the list
- On: grid shows owned items at full opacity + unowned items in sidebar-selected range at reduced opacity
- Sidebar filters apply to both owned and unowned items when toggle is on

### Missing Only Filter

- Toggle component in toolbar
- Off (default): shows all items
- On: shows only items where `complete === false`
- Combines with "Show Unowned": if both on, shows incomplete owned items + all unowned items

### Sidebar (List Detail)

- Same accordion pattern as catalog sidebar (expand DiddlType, click range)
- Additional: owned/total count displayed right-aligned on each range row as `text-xs text-muted-foreground`
- Count format: `{owned}/{total}` (e.g., "12/100")
- Toggle for showing counts: controlled by a small toggle icon button at sidebar top

## Accessibility Notes

- All toolbar actions must be keyboard-accessible (tab + enter/space)
- Dialog and AlertDialog handle focus trap and escape-to-close (shadcn-svelte built-in)
- Checkbox on cards uses proper `<input type="checkbox">` via shadcn Checkbox (ARIA compliant)
- Quantity stepper buttons have `aria-label="Decrease quantity"` and `aria-label="Increase quantity"`
- Color palette swatches use `role="radio"` with `aria-checked` and `aria-label="{color name}"`
- Toast announcements use `role="status"` (svelte-sonner default)
- "Show Unowned" and "Missing Only" toggles use `aria-pressed`
- Delete confirmation dialog uses `role="alertdialog"` (shadcn AlertDialog default)
- All interactive elements: minimum 44px touch target
- Edit icon button on list cards: `aria-label="Edit {listName}"`
- Delete icon button on list cards: `aria-label="Delete {listName}"`

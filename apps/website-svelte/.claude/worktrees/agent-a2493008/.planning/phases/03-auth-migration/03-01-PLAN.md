Create two new Convex function files implementing the server-side CRUD needed by ConvexListStore.

**src/convex/authed/lists.ts** — Four exports using `authedQuery` and `authedMutation` from `./helpers`:

1. `listByOwner` — authedQuery, no args. Queries `lists` table with `by_owner` index where `ownerSubject === ctx.identity.subject`. Returns all matching list docs. Returns the raw Convex documents (they have `_id`, `_creationTime`, `name`, `color`, `ownerSubject`).

2. `create` — authedMutation, args: `{ name: v.string(), color: v.string() }`. First checks list count via `by_owner` index — if `existing.length >= 3`, throw `new Error('List limit reached')` (per D-15, authed cap is 3). Then inserts into `lists` with `{ name: args.name, color: args.color, ownerSubject: ctx.identity.subject }`. Returns the new list ID.

3. `update` — authedMutation, args: `{ id: v.id('lists'), name: v.optional(v.string()), color: v.optional(v.string()) }`. Fetches the list by ID, verifies `ownerSubject === ctx.identity.subject` (throw `'Not authorized'` if not), then patches with provided fields using `ctx.db.patch(args.id, updates)` where `updates` is built from non-undefined args. Returns nothing.

4. `remove` — authedMutation, args: `{ id: v.id('lists') }`. Fetches list, verifies ownership. Deletes all listItems with `by_list` index where `listId === args.id`, then deletes the list itself. Returns nothing.

**src/convex/authed/listItems.ts** — Five exports:

1. `byList` — authedQuery, args: `{ listId: v.id('lists') }`. Verifies the list exists and is owned by current user. Queries `listItems` with `by_list` index. For each item, also fetches the linked `catalogItems` doc to include `type` and `number` in the response. Returns array of `{ ...listItemDoc, catalogType: catalogItem.type, catalogNumber: catalogItem.number }`.

2. `add` — authedMutation, args: `{ listId: v.id('lists'), items: v.array(v.object({ catalogItemId: v.id('catalogItems'), condition: v.union(v.literal('mint'), v.literal('near_mint'), v.literal('good'), v.literal('poor'), v.literal('damaged')), quantity: v.number(), complete: v.boolean(), tags: v.array(v.string()) })) }`. Verifies list ownership. For each item in args.items, inserts into `listItems` with `{ listId: args.listId, catalogItemId: item.catalogItemId, condition: item.condition, quantity: item.quantity, complete: item.complete, tags: item.tags }`. Returns array of new item IDs.

3. `remove` — authedMutation, args: `{ itemIds: v.array(v.id('listItems')) }`. For each ID, fetches the item, verifies the parent list is owned by current user, then deletes. Returns nothing.

4. `update` — authedMutation, args: `{ itemIds: v.array(v.id('listItems')), updates: v.object({ condition: v.optional(v.union(v.literal('mint'), v.literal('near_mint'), v.literal('good'), v.literal('poor'), v.literal('damaged'))), quantity: v.optional(v.number()), complete: v.optional(v.boolean()) }) }`. For each ID, fetches item, verifies parent list ownership, patches with provided updates. Returns nothing.

5. `duplicate` — authedMutation, args: `{ itemIds: v.array(v.id('listItems')) }`. For each ID, fetches the item and its parent list (verify ownership). Inserts a new listItem with same `listId`, `catalogItemId`, `condition`, `quantity`, `complete`, `tags`. Returns array of new item IDs.

Import `v` from `'convex/values'` and helpers from `'./helpers'` in both files.

Create `src/convex/authed/migration.ts` with a single export `migrateGuestData` using `authedAction` from `./helpers`.

Args:

```typescript
{
  guestSessionId: v.string(),
  lists: v.array(v.object({
    id: v.string(),        // guest UUID
    name: v.string(),
    color: v.string(),
  })),
  items: v.array(v.object({
    listId: v.string(),    // guest list UUID (maps to lists[].id above)
    type: v.string(),      // natural key part 1
    number: v.number(),    // natural key part 2
    condition: v.union(v.literal('mint'), v.literal('near_mint'), v.literal('good'), v.literal('poor'), v.literal('damaged')),
    quantity: v.number(),
    complete: v.boolean(),
    tags: v.array(v.string()),
  })),
}
```

Handler logic (all inside the `authedAction` handler which has `ctx` with `ctx.identity`):

1. **Idempotency check** (per CF-03): Query `migrations` table with `by_owner_session` index for `[ctx.identity.subject, args.guestSessionId]`. If a record exists, return early with `{ status: 'already_migrated', listsCreated: 0, itemsCreated: 0, itemsSkipped: 0 }`.

2. **Delegate to an internal mutation** for transactional writes. Use `ctx.runMutation(internal.authed.migration.executeMigration, { ... })` passing the full payload plus `ownerSubject: ctx.identity.subject`. The action cannot do transactional DB writes directly, so the mutation handles all inserts.

Also create an `internalMutation` export called `executeMigration` (using `internalMutation` from `'../_generated/server'`):

Args: same as above plus `ownerSubject: v.string()`.

Handler for `executeMigration`:

1. Check existing list count for this owner. If adding `args.lists.length` would exceed 3, only create enough to reach 3 (skip the rest).
2. For each guest list in `args.lists` (up to the cap): insert into `lists` with `{ name, color, ownerSubject: args.ownerSubject }`. Build a map: `guestListId -> convexListId`.
3. For each guest item in `args.items`:
   a. Look up `guestListId` in the map. If not found (list was skipped due to cap), skip the item. Increment `itemsSkipped`.
   b. Resolve `type:number` to `catalogItemId` by querying `catalogItems` with `by_type_number` index for `[item.type, item.number]`. If not found, skip the item and increment `itemsSkipped` (per Pitfall 3 — don't fail the whole migration).
   c. Check for duplicate via `by_list_catalog` index on `[convexListId, catalogItemId]`. If exists, skip and increment `itemsSkipped` (per D-08 dedup).
   d. Insert into `listItems` with `{ listId: convexListId, catalogItemId, condition: item.condition, quantity: item.quantity, complete: item.complete, tags: item.tags }`. Increment `itemsCreated`.
4. Insert a migration record into `migrations` table: `{ ownerSubject: args.ownerSubject, guestSessionId: args.guestSessionId, migratedAt: Date.now(), listsCreated, itemsCreated, itemsSkipped }`.
5. Return `{ status: 'migrated', listsCreated, itemsCreated, itemsSkipped }`.

The outer action (`migrateGuestData`) calls the internal mutation and returns its result.

Import `internal` from `'../_generated/api'` and `internalMutation` from `'../_generated/server'`. Import `v` from `'convex/values'`.

Run `bun run convex:gen` after creating the file.
